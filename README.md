## рҹ“Ң Project Overview

This is an **independently developed AI-driven Security Operations Center (SOC) framework** designed to automatically classify, prioritize, and respond to security alerts using machine learning. This project is entirely my own individual work.

It addresses one of the most critical challenges in modern SOC environments:

> вҡ пёҸ **Alert Fatigue** вҖ” overwhelming volumes of security alerts leading to delayed response and missed threats.

By leveraging a **Random Forest Classification Model** вҖ” selected after benchmarking four models against both synthetic and real-world data вҖ” this system significantly reduces manual analyst workload while improving detection accuracy and response efficiency. The model has been validated against **175,341 real-world network records** from the UNSW-NB15 benchmark dataset.

---

## рҹҡЁ Problem Statement

SOC analysts are inundated with thousands of alerts daily. The consequences are severe:

| Problem | Impact |
|--------|--------|
| вқҢ High false-positive rates | Analyst time wasted on non-threats |
| вқҢ Alert fatigue & burnout | Reduced effectiveness over time |
| вқҢ Delayed incident response | Threats go unaddressed for longer |
| вқҢ Inconsistent prioritization | Critical alerts may be missed |
| вқҢ Not scalable manually | Cannot keep up with growing data volumes |

> Manual triage is time-consuming, error-prone, and unsustainable at enterprise scale.

---

## рҹ’Ў Solution Architecture

A **Random Forest Classification Model** trained on simulated SOC alert data that:

### рҹ”№ Core Capabilities

- рҹ”Қ Classifies alerts into **Critical / High / Medium / Low** severity
- вҡЎ Automatically prioritizes threats in real time
- рҹӣЎпёҸ Recommends targeted remediation actions per severity
- рҹ“Ҡ Measures and models analyst workload reduction
- рҹ”Қ Provides feature importance analysis for transparency
- вң… **Validated against real-world UNSW-NB15 benchmark data**

## рҹҢҚ Real-World Validation вҖ” UNSW-NB15 Dataset

### Key Findings

- вң… **100% accuracy on 175,341 real-world records** вҖ” the model generalises effectively beyond synthetic training data
- рҹ“ү **Workload reduction of 31.3%** on real data vs 42.5% on synthetic вҖ” real-world traffic contains a higher proportion of serious threats requiring human review, which is the **correct and expected behaviour** for a production SOC triage system
- рҹ”ҙ The model correctly identified all 196 Critical alerts in the real dataset with near-perfect precision and recall
- рҹ§Ә Trained on 82,332 real records, tested on 175,341 вҖ” demonstrating strong scalability

> вң… Real-world validation confirms this framework is viable for production SOC deployment at scale.

## рҹ§Ә Model Comparison вҖ” Four Algorithms Benchmarked

Random Forest was not chosen arbitrarily. Four models were trained and evaluated on both synthetic and real-world data to justify the selection.

### Results вҖ” Synthetic Data (1,000 records)

| Model | Accuracy (%) | Training Time (s) |
|-------|-------------|------------------|
| рҹҘҮ Random Forest | **98.0%** | 0.379s |
| рҹҘҲ XGBoost | 98.0% | 0.355s |
| рҹҘҲ Decision Tree | 98.0% | 0.005s |
| рҹҘү Logistic Regression | 94.0% | 0.980s |

### Results вҖ” Real Data (175,341 records вҖ” UNSW-NB15)

| Model | Accuracy (%) | Training Time (s) |
|-------|-------------|------------------|
| рҹҘҮ Random Forest | **100.00%** | 1.78s |
| рҹҘҲ Decision Tree | 100.00% | 0.07s |
| рҹҘҲ XGBoost | 99.99% | 1.59s |
| рҹҘү Logistic Regression | 99.88% | 41.02s |

### Full Comparison Table

| Model | Synthetic Acc (%) | Synthetic Time (s) | Real Acc (%) | Real Time (s) |
|-------|------------------|-------------------|--------------|---------------|
| Random Forest | 98.0% | 0.379s | **100.00%** | 1.78s |
| Decision Tree | 98.0% | 0.005s | 100.00% | 0.07s |
| XGBoost | 98.0% | 0.355s | 99.99% | 1.59s |
| Logistic Regression | 94.0% | 0.980s | 99.88% | 41.02s |

### вң… Why Random Forest Was Selected

Despite Decision Tree matching Random Forest's accuracy on this dataset, Random Forest was selected as the production model for the following reasons:

| Reason | Detail |
|--------|--------|
| **Overfitting resistance** | A single Decision Tree memorises training data вҖ” 100 trees voting together generalises far better to unseen threats |
| **Interpretability** | Feature importance charts show analysts exactly which signals drive each decision вҖ” critical for SOC trust and auditability |
| **Robustness** | Performance stays consistent across both clean synthetic data and noisy real-world traffic |
| **No feature scaling needed** | Works directly with mixed numeric and binary security features |
| **Industry standard** | Widely deployed in production SIEM and SOAR integrations |

### вқҢ Why Logistic Regression Was Eliminated

- Assumes linear relationships between features вҖ” security alert data is not linearly separable
- 94% accuracy on synthetic data is significantly lower than the other models
- 41 seconds training time on real data makes it impractical at enterprise scale

## рҹӣЎпёҸ Intelligent Remediation Engine

Each predicted severity level automatically triggers targeted response recommendations:

| Severity | Level | Recommended Actions |
|----------|-------|---------------------|
| рҹ”ҙ **Critical** | P1 вҖ” Immediate | Isolate host В· Block IP В· Reset credentials В· Escalate immediately to Tier 2/3 |
| рҹҹ  **High** | P2 вҖ” Urgent | Investigate IP В· Reset user credentials В· Analyze authentication logs |
| рҹҹЎ **Medium** | P3 вҖ” Moderate | Monitor activity closely В· Review user behaviour patterns |
| рҹҹў **Low** | P4 вҖ” Routine | Log event В· Review during next scheduled audit cycle |
